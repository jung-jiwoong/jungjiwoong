Problem Solving

실제 업무에서 마주한 문제들을 어떻게 해결했는지 소개합니다.

모니터링 시스템 부재로 인한 장애 대응 지연 문제

Situation 상황

기존에는 장애가 발생해야 고객 민원을 통해 인지하는 구조였습니다. 장비 상태를 실시간으로 확인할 방법이 없어 장애 대응이 항상 늦어졌고, 원인 분석에도 많은 시간이 소요되었습니다.

Task 과제

장애를 사전에 감지하고, 발생 시 빠르게 원인을 파악할 수 있는 모니터링 시스템을 구축해야 했습니다.

Action 행동

IPSec 터널을 맺어 모니터링용으로 구축하였으며 Automation 기능을 활용하여 장비에 발생한 이벤트 또는 CPU, 메모리, 디스크 사용량이 임계치를 넘으면 메신저로 알림을 받도록 Slack과 이메일로 즉시 알림이 가도록 설정했습니다.

Result 결과

장애 징후를 미리 감지하여 선제적 대응이 가능해졌고, 장애 발생 시에도 원인을 빠르게 파악할 수 있게 되었습니다.

VPN을 통한 웹 접속 시에만 “연결할 수 없음” 오류 발생 문제

Situation 상황

Cisco 라우터 기반 IPsec VPN 환경에서 상대방 클라이언트 장비 내부 서버로의 웹 접속이 불가능한 장애가 발생했습니다. 외부 인터넷에서는 정상 접속이 가능했으나 VPN을 통한 접속 시에만 “연결할 수 없음” 오류가 발생하는 상황이었습니다.

Task 과제

VPN 환경에서만 발생하는 웹 접속 장애의 원인을 분석하고 정상적인 통신이 가능하도록 문제를 해결하는 것이 목표였습니다.

Action 행동

- ping, tcping 및 디버그 테스트를 통해 기본 네트워크 통신은 정상임을 확인

- 웹 접속 오류 원인을 분석하기 위해 Wireshark 패킷 분석 수행

- 패킷 분석 결과 TCP 세그먼트 손실 현상 발생 확인

- VPN 환경에서만 문제가 발생하는 점에 주목하여 IPsec 터널링 시 암호화 및 캡슐화 오버헤드로 인한 MTU 초과 가능성 조사

- 테스트 장비에서 MTU 값을 조정하여 테스트 진행

Result 결과

MTU 값을 조정한 이후 VPN 환경에서도 웹 접속이 정상적으로 이루어졌으며, 동일한 MTU 설정을 VPN 장비에도 적용하여 문제를 최종적으로 해결했습니다.
